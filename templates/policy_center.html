Policy Setup Guidelines

Each policy setting below is now written as an operator guide: What it controls, recommended setup, and risk note. Manual edits save as policies.mode=custom. Use Dry Run after changing policies before enabling scheduler or auto-apply.

preserve_rowsKeep existing rows. Safest when source data may be temporarily unavailable.
warn_onlyShow warning but do not delete. Useful while tuning policies.
cleanup_immediateDelete stale rows in the same sync cycle. Fastest but can increase apply frequency.
cleanup_next_runMark stale now and delete on next successful run. Retained for custom compatibility; Singularity normally uses immediate cleanup only after trusted source scans.
require_confirm_next_runAsk operator confirmation, then apply cleanup on next run. Safest for risky changes.
block_cleanup / block_applyStop deletion or LibreQoS apply when policy detects unsafe conditions.
